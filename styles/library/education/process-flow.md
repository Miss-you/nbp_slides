
## 风格描述
工程蓝图流程图风格以传统工程制图蓝图（Blueprint）为视觉基底，将算法逻辑流、系统架构图和标准化流程以高精度技术图纸的方式呈现。深蓝色背景模拟晒图纸（Cyanotype）的经典质感，所有图形元素用白色或浅蓝线条绘制，与背景形成鲜明的专业技术对比。流程图严格遵循国际标准符号规范：菱形代表判断/决策节点，矩形代表执行/动作步骤，圆角矩形代表开始/结束节点，平行四边形代表输入/输出。连接符统一使用直角折弯箭头（Right-angle Elbow Arrows），所有连线保持水平或垂直走向，严禁使用斜线。网格辅助线作为隐性规范强化了技术精准感。这一风格适合算法讲解、系统设计文档、工程工艺流程说明、代码逻辑可视化及任何需要展示高技术专业度的内容场景。

## 基础提示词模板

你是一位专业的工程信息设计师，请生成16比9的工程蓝图流程图风格演示文稿幻灯片。

**视觉风格与美术指导**

背景风格：使用深蓝色蓝图背景（Blueprint Aesthetic）。背景底色为深普鲁士蓝 #0D2137 或深海军蓝 #0A192F。在背景上叠加细网格线（Grid Lines），网格线颜色为略亮的蓝色 #1A4A7A 或 #1E3A5F，透明度30-40%，间距均匀（约40-50px等效）。禁止使用纯黑背景或白色背景，必须保持蓝图的标志性深蓝调。

主图形元素（严格遵循ISO/ANSI流程图符号标准）：
- 开始/结束节点：圆角矩形（Rounded Rectangle），比例约3:1（宽:高），白色边框2-3px，内部填充半透明深蓝 #1A4A7A，文字白色；
- 执行/动作步骤：标准矩形（Rectangle），比例约4:2（宽:高），白色实线边框2px，内部填充半透明深蓝，文字白色，字号适中；
- 判断/决策节点：菱形（Diamond），四角对齐上下左右方向，白色实线边框2px，内部填充透明或半透明，文字白色，YES/NO分支标注在连出箭头旁；
- 输入/输出节点：平行四边形（Parallelogram），倾斜角约15度，白色实线边框2px，内部半透明填充，文字白色；
- 子流程节点（可选）：带两侧竖线的矩形，表示引用外部子程序或子流程。

连接符规范（Critical — 严格执行）：
所有连线必须使用直角折弯箭头（Right-angle Elbow Arrow），即连线仅走水平（→←）和垂直（↑↓）方向，在需要转向时以90度直角折弯，严禁出现任何斜向连线。箭头末端为实心等边三角形。连线颜色为白色 #FFFFFF 或浅蓝 #7EC8E3，线宽1.5-2px。分支标注（YES/NO 或 是/否）置于连线靠近菱形出口的位置，字号小，颜色亮黄 #FFD700 或白色。

辅助标注元素：
- 步骤编号：在每个图形左上角或外侧添加小型圆形编号标签（填色为橙色 #FF8C00 或亮蓝 #00B4D8，白色数字）；
- 泳道区域（可选，用于多角色流程）：使用垂直或水平的半透明浅蓝色条带划分不同执行主体的泳道，泳道标签置于左侧或顶部；
- 注释框：使用折角矩形（带折角的便签形状），填色浅黄半透明，文字深色，用虚线连接到对应节点；
- 标题框：页面左上角或右上角放置带蓝图式边框的标题栏（类似工程图纸的标题栏格式），内置幻灯片标题、编号和版本号。

**配色方案**

主背景色：深普鲁士蓝 #0D2137
次背景/网格线：深蓝 #1A4A7A（透明度35%）
主图形边框/连线：白色 #FFFFFF
图形填充色（半透明）：#1A4A7A（透明度50%）
决策节点高亮填充：深青蓝 #005F8A（透明度60%）
YES分支标注：亮绿 #4ECB71
NO分支标注：亮红橙 #FF6B6B
步骤编号标签：橙色 #FF8C00
重要流程高亮：亮蓝 #00B4D8
注释/说明色：亮黄 #FFD700
文字色（图形内）：纯白 #FFFFFF
标题栏底色：深蓝 #0A2744

**字体排版**

图形内步骤文字：清晰等宽无衬线体或几何感无衬线体，字号适中（图形内居中对齐），白色，粗细适当（避免过细导致在深色背景上难以阅读）。
分支标注（YES/NO）：小号粗体，亮绿/亮红色，位于连线旁。
步骤编号：粗体白色数字，置于彩色圆形标签内。
页面标题（标题栏内）：等宽体或无衬线粗体，白色，左对齐。
注释文字：小号无衬线常规体，深色（置于浅色注释框内）。

## 页面类型模板

### 封面页模板
构图逻辑：深蓝色蓝图背景，叠加均匀网格线。页面中央绘制一个简化的"总览流程图"——以开始到结束的高层级流程框架呈现，图形数量控制在5-7个，展示整体流程的逻辑骨架。页面右下角放置工程图纸式标题栏（矩形框内）：标题（大号粗体白色）、副标题、编号，仿照技术图纸格式。左上角可添加小型装饰性网格坐标轴或比例尺元素，强化蓝图质感。

使用场景：PPT第一页，算法/系统/工程流程的总览展示。

### 内容页模板
构图逻辑：深蓝色背景加网格线。主体区域放置详细流程图，包含完整的开始节点、执行步骤矩形、判断菱形（含YES/NO分支）和结束节点，所有连线使用直角折弯箭头，严格水平/垂直走向。步骤编号圆形标签标注于每个图形外侧。关键步骤或判断节点可使用亮蓝 #00B4D8 边框高亮强调。复杂流程可添加泳道划分不同执行主体。注释框配虚线连接至对应节点提供额外说明。

使用场景：详细展示算法逻辑、业务流程、系统工作流、决策树等。

### 数据页模板
构图逻辑：深蓝色背景，将数据信息以技术图表形式呈现——使用蓝图风格的折线图、柱状图或热力表（所有图表元素使用白色/浅蓝线条，配色方案保持深蓝底白线风格）。数据图表配合小型流程图节点图标（矩形/菱形）标注数据分类。关键数字用橙色编号标签或亮蓝高亮框放大展示。图表右下角保留工程图纸标题栏格式，注明数据来源和日期。

使用场景：展示系统性能指标、算法比较、流程效率数据、技术参数对比。

### Nano Banana Pro配置
- 模型：gemini-3-pro-image-preview
- 比例：16:9
- 分辨率：2K (2752x1536) 或 4K (5504x3072)

### 建议设置
- 推荐分辨率：2K（平衡质量和生成速度）
- 适用于：算法与逻辑流程讲解、系统架构文档、工程工艺流程说明、标准作业程序（SOP）、软件开发流程展示、决策树分析、技术培训材料等场景
